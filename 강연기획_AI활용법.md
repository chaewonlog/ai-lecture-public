# 강연 기획 정리
> ChatGPT 매일 쓰는데 왜 나만 결과가 별로지?

---

## 기본 정보

- **제목**: ChatGPT 매일 쓰는데 왜 나만 결과가 별로지?
- **부제**: AI 초보가 고수처럼 쓰는 3가지 방법
- **장소**: 연남서가 (오프라인 + 온라인 동시 송출)
- **시간**: 60분
- **대상**: AI 써보고 싶은데 어디서 시작할지 모르는 분 / 쓰고 있는데 결과가 아쉬운 분

---

## 커리큘럼

### 오프닝 (5분)
- 청중 손들기: "AI 써보신 분?" → "매일 쓰시는 분?" → "만족하시는 분?"
- 슬라이드 한 장 — 격차 체감
  - 블로그 초안: 3시간 → 30분
  - 회계감사보고서 분석: 3~5일 → 2시간
- "차이는 도구가 아니라 방법입니다"

---

### 1️⃣ AI한테 나를 기억시켜라 (10분)
**— 매번 설명하는 사람 vs 한 번만 설명하는 사람**

- "AI는 핸드폰 기종 고르는 것과 같아요. 써봐야 내 손에 맞는 게 뭔지 알아요. 저도 Gemini 쓰다가 Claude로 넘어왔고, 지금은 둘 다 써요. 오늘 세 가지 다 써볼 거니까 끝나고 제일 마음에 드는 걸로 시작하세요." (1분)
- ChatGPT 커스텀 인스트럭션 라이브 세팅 시연 (3분)
- 환각 차단 시연 (4분)
  - 질문: "2026년 한국 소상공인 종사자 수가 몇 명이야?"
  - 인스트럭션 없으면: 그럴듯한 숫자 자신 있게 말함
  - 인스트럭션 있으면: "확실하지 않습니다, 출처 확인 필요"
  - 라이브 비교 or 캡처 백업
- "집에 가서 바로 복붙하세요" (1분)
- **배포**: 타입별 커스텀 인스트럭션 템플릿

---

### 2️⃣ 잘 된 거 뜯어서 내 걸로 만들어라 (40분)
**— 프롬프트 잘 짜려고 고민하지 마세요. 구조는 훔치고 내용만 바꾸면 됩니다**

3단계 프레임 한 장으로 먼저 제시 (2분)
- **진단**: 뭐가 잘 됐는지 먼저 파악
- **역설계**: 어떤 프롬프트였는지 추출
- **검증 → 이식**: 돌려보고 내 변수로 교체

#### 시연 1 — 프롬프트 역설계 (18분)
- 프롬프트 사이트에서 잘 된 결과물 하나 고르기
- 진단 → 역설계 → 검증 → 내 변수로 이식 라이브 시연
- "이게 되면 어디든 똑같이 적용됩니다"
- **배포**: 역설계 3단계 프롬프트 템플릿

#### 시연 2 — 나만의 봇 만들기 (18분)
- GPTs(ChatGPT) / Gems(Gemini) 중 하나 라이브 세팅
- 시연 1에서 역설계한 프롬프트를 봇에 그대로 심기
- "이제 매번 프롬프트 안 써도 됩니다"
- **배포**: 봇 세팅용 프롬프트 템플릿

---

### 마무리 + Q&A (10분)
- 두 가지 방법 한 장 요약
- 숙제: "집에 가서 내가 자주 쓰는 프롬프트 하나 골라서 봇으로 만들어보세요"
- 심화 워크샵 관심자 QR 등록 안내
- Q&A

---

### 보너스 — 시간 되면 (5분)
**"AI가 알아서 뉴스 브리핑해줍니다"**
- Make.com 자동화 플로우 캡처 + 실제 결과물 공개
- "한 번 세팅하면 매일 자동으로 옵니다"

## 타입별 커스텀 인스트럭션 템플릿

### 타입 1 — 환각 차단형
> 틀린 정보가 싫은 사람

```
아래의 규칙을 엄격히 준수하여 답변해 주세요.
1) 충분한 근거가 없거나 정보가 불확실한 경우, 절대 임의로 지어내지 말고 "알 수 없습니다" 또는 "잘 모르겠습니다"라고 명시해 주세요.
2) 답변하기 전 단계별로 정보를 검증하고, 모호하거나 출처가 불분명한 부분은 "확실하지 않음"이라고 표시하세요.
3) 최종적으로 확실한 정보만 사용하여 간결한 답변을 완성하세요. 추측이 불가피할 경우 "추측입니다"라고 밝혀주세요.
4) 문의가 모호하거나 추가 정보가 필요하다면 먼저 맥락이나 세부 정보를 요청하세요.
5) 확인되지 않은 사실을 단정 짓지 말고, 필요한 경우 근거를 함께 제시하세요.
6) 출처나 근거가 있는 경우 해당 정보를 명시하고, 가능하면 관련 링크나 참고 자료를 간단히 요약해 알려주세요.
```

### 타입 2 — 글쓰기 특화형
> 블로그, SNS, 뉴스레터 자주 쓰는 사람

```
글쓰기 작업 시 아래 규칙을 따라주세요.
1) 불필요한 서론, 배경 설명 없이 바로 본문으로 시작하세요.
2) 문어체가 아닌 자연스러운 구어체로 작성하세요.
3) 문장은 짧고 명확하게, 단락은 3줄 이내로 유지하세요.
4) 강조나 핵심 메시지는 첫 문장 또는 마지막 문장에 배치하세요.
5) 독자가 끝까지 읽고 싶게 만드는 훅(hook) 문장을 첫 줄에 넣어주세요.
6) 이모지나 과도한 bullet point 사용은 피하세요.
```

### 타입 3 — 업무 효율형
> 보고서, 정리, 요약 자주 하는 사람

```
업무 관련 답변 시 아래 규칙을 따라주세요.
1) 결론을 먼저 말하고, 근거나 설명은 그 다음에 제시하세요.
2) 답변은 bullet point로 간결하게 정리하세요.
3) 불필요한 인사말, 감탄사, 마무리 멘트는 생략하세요.
4) 숫자나 데이터가 있으면 반드시 포함하세요.
5) 애매한 표현 대신 구체적이고 실행 가능한 표현을 사용하세요.
6) 요청이 모호하면 명확히 해달라고 먼저 물어보세요.
```

## 강연 후 연결 구조

```
연남서가 강연 (무료 or 소액)
↓
관심자 QR 등록
↓
① 유료 템플릿 (Publy / 크몽 / Gumroad)
② 소규모 심화 워크샵
③ 1:1 컨설팅
```

### 유료 템플릿 후보

| 템플릿 | 설명 | 추천 플랫폼 |
|--------|------|------------|
| 환각 차단 인스트럭션 세트 | 타입별 3종 | Publy |
| 역설계 3단계 프롬프트 세트 | 진단/역설계/이식 | Publy / 크몽 |
| 봇 세팅 가이드 | GPTs / Gems | 크몽 |
| Make.com 뉴스 자동화 가이드 | 설정 방법 전체 | Gumroad |
