# 프롬프트 잘 짜는 메타 체크리스트

> 강연 배포물. 내 프롬프트를 이 체크리스트로 한 번 걸러보세요.
> AI 답변 품질이 바로 올라갑니다.

---

## 왜 이 체크리스트인가

"AI야, 이거 해줘" 수준의 프롬프트는 **뻔한 답**만 뱉습니다.

좋은 프롬프트는 6가지 요소를 포함합니다. 이 6가지를 매번 체크하면 결과 품질이 올라가요.

---

## 좋은 프롬프트의 6가지 메타 요소

### 1. 목적을 먼저 말한다

**BEFORE**
```
블로그 글 써줘
```

**AFTER**
```
저는 AI 실험 로그 뉴스레터를 운영합니다.
이번 주 발행할 블로그 오프닝을 써주세요.
```

- AI에게 **"이 대화의 목적이 뭔지"** 먼저 말함
- 내 역할·상황·의도를 첫 문장에 담음

---

### 2. 데이터를 넣는다 (추상 → 구체)

**BEFORE**
```
우리 제품 랜딩페이지 분석해줘
```

**AFTER**
```
우리 제품 랜딩페이지 분석해주세요.
- URL: ...
- 방문자 수: 월 1만명
- 전환율: 1.2%
- 목표: 전환율 3%로 끌어올리기
- 우리 타겟: 30대 스타트업 PM
```

- 숫자·맥락·목표를 명시
- AI는 추상적 조언 대신 **구체적 제안**을 뱉음

---

### 3. 성공·실패 사례를 같이 준다

**BEFORE**
```
이런 뉴스레터 제목 좋은 거 뽑아줘
```

**AFTER**
```
아래 사례를 참고해서 제목 5개 뽑아주세요.

[성공 사례] (조회수 포함)
- "연말 허무함은 성과가 없어서가 아니라 기록이 없어서입니다" (3,570)
- "비개발자인 내가 SQL 비서를 만든 이야기" (1,460)

[실패 사례 · 반면교사]
- "IDE를 쓰기 시작한 진짜 이유" (88)
- "잘하는 PM의 정의는 회사 나이테마다 다릅니다" (244)
```

- **성공만 ❌**
- **성공 + 실패 둘 다 ⭕** → AI가 "결정적 차이"를 뽑아냄

---

### 4. 원하는 패턴에 이름을 붙인다

**BEFORE**
```
재밌고 눈에 띄는 제목으로 써줘
```

**AFTER**
```
아래 3가지 공식 중 하나로 써주세요.

- 반전형: "A가 아니라 B" 구조
- 변환형: "X를 Y로 바꾸는" 구조
- 선언형: "저는 ~합니다" 구조

각 공식별로 1개씩 후보 주세요.
```

- 원하는 스타일에 **기억나는 이름** 부여
- AI가 정확히 그 스타일로 뱉음

---

### 5. 피해야 할 것을 명시한다 (함정 리스트)

**BEFORE**
```
자연스럽게 써줘
```

**AFTER**
```
자연스럽게 써주세요. 단, 피해야 할 것:
- "디지털 시대에", "바야흐로", "우리 모두가 주목해야 할" 같은 클리셰
- 서론으로 "~에 대해 알아보겠습니다"
- 문어체 ("~할 수 있습니다" / "~라고 할 수 있습니다")
- 이모지 남발
- 한 문장 두 줄 넘기기
```

- "잘 써줘" = 애매
- "이거 피해라" = **명확한 가이드**

---

### 6. 원하는 출력 형식을 준다

**BEFORE**
```
요약해줘
```

**AFTER**
```
아래 형식으로 요약해주세요.

​```
결론: [한 줄]

근거:
1. ...
2. ...
3. ...

다음 액션: [한 줄]
​```

- 결론 먼저, 근거 3개, 다음 액션으로 끝냄
- 인사말·맺음말 생략
```

- AI가 **따를 정확한 템플릿** 제공
- 매번 결과 형식이 흔들리지 않음

---

## 프롬프트 품질 체크리스트 (14개)

프롬프트를 보낼 때 이 체크리스트를 통과하는지 확인.

### 맥락 (3개)
- [ ] 내 역할·상황을 첫 문장에 명시했나
- [ ] 구체적 데이터·숫자·맥락을 포함했나
- [ ] AI가 모를 수도 있는 배경 설명 있나

### 예시 (3개)
- [ ] 성공 사례 3개 이상 포함했나
- [ ] 실패 사례(반면교사)를 같이 줬나
- [ ] 원하는 스타일에 이름을 붙였나

### 제약 (4개)
- [ ] 피해야 할 것을 명시했나 (함정 리스트)
- [ ] 분량·길이 제한을 줬나
- [ ] 톤·무드를 구체적으로 정했나
- [ ] 기술용어·일반용어 구분을 했나

### 출력 (4개)
- [ ] 원하는 출력 형식 템플릿을 줬나
- [ ] 결과 개수를 명시했나 (3개? 5개?)
- [ ] 실행 프로세스 순서를 줬나
- [ ] 자체 검증 요청을 넣었나 ("다시 한 번 확인")

## 나쁜 프롬프트 vs 좋은 프롬프트 전환 예시

### 예시 1 · 회의 요약

**❌ 나쁜 프롬프트**
```
이 회의록 요약해줘.
[회의록 본문]
```

**⭕ 좋은 프롬프트**
```
저는 PM이고, 팀장한테 보고할 회의 요약이 필요합니다.

[회의록 본문]

요약 조건:
- 분량: 5줄 이내
- 형식:
  결론 한 줄 / 합의된 사항 3개 불릿 / 다음 액션 1~2개
- 톤: 경어체, 단정적
- 피해야 할 것: 장황한 설명, 인사말, "~할 예정입니다" 같은 미완결 어미

참고 — 전에 팀장이 좋다고 했던 보고 스타일:
"결론: A안 선택. 이유 3개. 다음 액션: B부서와 협의"
```

---

### 예시 2 · 이메일 답장

**❌ 나쁜 프롬프트**
```
고객한테 환불 거절 메일 보내야 돼. 정중하게 써줘.
```

**⭕ 좋은 프롬프트**
```
저는 SaaS 고객 운영 담당입니다.
고객의 환불 요청을 정책상 거절해야 하는 상황이에요.

[상황]
- 고객: 1년 약정 3개월 차
- 요청: 전액 환불
- 우리 정책: 3개월 이후 환불 불가

메일 조건:
- 분량: 4~5줄
- 구조: 공감 → 정책 설명 → 대안 제안 → 마무리
- 톤: 정중하지만 단호
- 피해야 할 것: "죄송합니다만"으로만 시작, 책임 회피 문구,
              "양해 부탁드립니다" 같은 공허한 표현

대안 예시: 1개월 연장 or 크레딧 전환 제안.
```

## 한 줄 요약

**목적 + 데이터 + 성공·실패 사례 + 패턴 이름 + 함정 + 출력 형식**

이 6가지를 프롬프트에 매번 넣으면 AI 결과 품질이 흔들리지 않습니다.

---

## 더 깊이 쓰고 싶다면

이 6가지가 익숙해지면, 다음 단계는 **"반복 개선 프롬프트"**:

```
방금 결과물을 앞서 정한 공식에 비춰 자체 평가해주세요.
1. 어떤 공식을 적용했는가
2. 어느 요소가 약한가
3. 함정 중 저지른 것이 있는가

분석 바탕으로 다시 써주세요.
```

- AI가 **자기 결과를 스스로 평가**하게 만듦
- 메타 분석 후 재작성 → 품질 한 단계 더 올라감
