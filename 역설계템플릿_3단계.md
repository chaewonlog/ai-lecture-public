# 역설계 3단계 프롬프트 템플릿

> 강연 배포물. 복붙해서 바로 쓰세요.
> ChatGPT / Claude / Gemini 모두 동일하게 작동합니다.

---

## 핵심 원리

**프롬프트 잘 짜려고 고민하지 마세요. 잘 된 결과물이 이미 넘쳐납니다.**

1. **진단** — 잘 된 결과물 + 실패 사례 모으기
2. **역설계** — AI한테 "공식북 만들어줘" 요청
3. **이식** — 뽑아준 공식을 내 변수로 교체

---

## STEP 1 · 진단 (사례 모으기)

### 어디서 찾을까
- **블로그 · 뉴스레터** → Medium, Brunch, 읽어본 좋은 뉴스레터
- **카드뉴스 · SNS** → 인스타 비즈니스 계정 (조회·저장 많은 것)
- **유튜브** → 조회수 높은 영상의 썸네일·제목
- **PPT · 랜딩페이지** → SlideShare, 노션 공개 템플릿, 전환 잘 된 랜딩
- **보고서 · 기획안** → 상사에게 "통과된" 것
- **CS 응대** → 브랜드의 좋은 응대 메시지 캡처

### 얼마나 모을까
- **성공 사례 5~10개** + **실패 사례 3~5개**
- 실패 사례(반면교사)가 있어야 AI가 "**결정적 차이**"를 뽑아냄
- 성공만 모으면 AI가 표면적 패턴만 뽑음

### 주의사항
- "내가 좋다고 느낀 것"으로 모으세요
- 가능하면 **성과 지표** 함께 (조회수, 전환율, 좋아요 수 등)
- 전문가 추천보다 본인 취향에 맞는 것

---

## STEP 2 · 역설계 (공식북 뽑기)

### 복붙 가능한 메인 프롬프트

```
아래 [결과물 종류]의 성공·실패 사례를 분석해서
"공식북"을 만들어주세요.

[성공 사례 5~10개] (성과 지표 포함)
사례 1: ... (조회수: ___)
사례 2: ... (조회수: ___)
...

[실패 사례 3~5개] (반면교사)
사례 1: ... (왜 안 먹혔나: ___)
...

---

출력 요구사항:

1. 공식 TOP 5
   - 각 공식에 기억나는 이름 부여
     (예: "반전형", "변환형", "1인칭 선언형")
   - 각 공식의 구조 + 적용 예시 2~3개

2. 성공 vs 실패의 결정적 차이 3가지

3. 함정 5가지 (피해야 할 것)
   - 구체적 표현·구조·장르 명시

4. 실행 프로세스
   - 내가 직접 쓸 때의 단계별 순서

표·번호 형태로 정리. 뻔한 분석 말고
제가 직접 분석했다면 놓쳤을 패턴 위주로 뽑아주세요.
```

### 왜 이렇게 강하게 짜야 하나

**약한 프롬프트** ("패턴 뽑아줘") → AI가 일반적·표면적 분석만 함
**강한 프롬프트** (위) → AI가 다음 6가지를 강제로 수행:
- 데이터 기반 (성과 지표)
- 성공·실패 대비 (반면교사)
- 패턴 네이밍 (기억하기 쉬움)
- 함정 명시 (피해야 할 것)
- 실행 프로세스 (바로 적용)
- 출력 구조화 (표·번호)

→ 결과물 품질이 흔들리지 않습니다.

---

## STEP 3 · 이식 (내 변수로 교체)

### 복붙 가능한 프롬프트

```
위 공식북을 바탕으로, 아래 조건에 맞춰
[결과물 종류]를 작성해주세요.

- 주제: [내 주제]
- 독자 / 타겟: [누구를 위한 것인가]
- 톤 · 무드: [원하는 느낌 — 예: 친근한, 전문적인, 유머러스한]
- 분량: [예: 3문단 / 500자 이내 / 10줄]
- 꼭 포함할 것: [핵심 메시지 · 키워드 · 사례]
- 피해야 할 것: [클리셰 · 특정 표현 · 형식]
- 적용할 공식: [공식북에서 고른 것 — 예: "반전형 + 1인칭 선언형"]

위 조건과 공식을 충실히 적용해서 작성해주세요.
함정 5가지는 반드시 회피.
```

### 결과물이 아쉬울 때 (반복 개선) ⭐

```
방금 결과물을 공식북에 비춰 자체 평가해주세요.

1. 어떤 공식을 적용했는가
2. 어느 요소가 약한가
3. 함정 중 저지른 것이 있는가

분석 바탕으로 다시 써주세요.
이번엔 [특정 공식]을 가장 강하게 적용해서.
```

→ AI가 **자기 결과를 스스로 평가**하게 만든 후 재작성. 메타 분석 → 재시도 구조라 품질이 한 단계 더 올라감.

---

## 활용 예시 6가지

### 1. 블로그 · 뉴스레터 (문체 · 전개)
- 진단: 좋아하는 뉴스레터 최근 글 5편 + 조회수 낮았던 글 3편
- 역설계: 공식북 (오프닝·문단 호흡·구어체 패턴 + 함정)
- 이식: 이번 주 내 주제로 공식 적용

### 2. 카드뉴스 · SNS (훅 · 구성)
- 진단: 저장 많은 카드뉴스 5개 + 무반응 3개
- 역설계: 1장 훅 공식 · 슬라이드 순서 패턴 · 마지막 CTA 패턴
- 이식: 내 브랜드 콘텐츠 기획

### 3. 유튜브 (썸네일 · 제목)
- 진단: 내 채널/동종 채널 조회수 TOP 10 + BOTTOM 5
- 역설계: 제목 공식 · 썸네일 텍스트 배치 · 감정 단어 사용
- 이식: 다음 영상 제목 · 썸네일

### 4. PPT · 랜딩페이지 (레이아웃 · CTA)
- 진단: 전환 잘 된 랜딩 3개 + 전환 안 된 랜딩 2개
- 역설계: 섹션 순서 · CTA 배치 · 헤드라인 공식
- 이식: 내 서비스 랜딩 구조

### 5. 보고서 · 기획안 (구조 · 근거 배치)
- 진단: 내 팀에서 통과된 보고서 3~5개 + 반려된 보고서 2개
- 역설계: 결론 위치 · 근거 순서 · 데이터 제시 공식
- 이식: 이번 주 보고서 작성

### 6. CS · 고객 응대 (말투 · 해결 플로우)
- 진단: 잘한 CS 응대 캡처 5개 + 못한 것 2개 (반면교사)
- 역설계: 공감 → 원인 → 해결 → 다음 액션 공식, 호칭, 마무리 문장
- 이식: 이번 주 CS 이슈 응대

## 자주 하는 실수 · 피하세요

| 실수 | 이유 | 대안 |
|---|---|---|
| 성공 사례만 모으기 | AI가 표면적 패턴만 뽑음 | **실패 사례 같이 넣기** (반면교사) |
| "패턴 뽑아줘" 정도 약한 프롬프트 | AI가 뻔한 답만 냄 | 공식 네이밍·함정·프로세스까지 요구 |
| "잘 써줘" 같은 모호한 이식 요청 | AI가 일반적 답만 냄 | 주제·독자·톤·분량 구체 지정 |
| 한 번에 완벽한 결과 기대 | 첫 결과는 보통 70점 | **자체 평가 후 재작성** 프롬프트로 90점 |
| 패턴만 보고 내 맥락 무시 | 복붙 결과물이 됨 | 내 변수(상황·독자·브랜드)로 항상 교체 |
