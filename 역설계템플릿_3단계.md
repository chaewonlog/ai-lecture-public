# 역설계 3단계 프롬프트 템플릿

> 강연 배포물. 복붙해서 바로 쓰세요.
> ChatGPT / Claude / Gemini 모두 동일하게 작동합니다.

---

## 핵심 원리

**프롬프트 잘 짜려고 고민하지 마세요. 잘 된 결과물이 이미 넘쳐납니다.**

1. **진단** — 잘 된 결과물 3~5개 모으기
2. **역설계** — AI한테 "공통 패턴 뽑아줘" 요청
3. **이식** — 뽑아준 패턴을 내 변수로 교체

---

## STEP 1 · 진단 (잘 된 결과물 모으기)

### 어디서 찾을까
- **블로그 · 뉴스레터** → Medium, Brunch, 읽어본 좋은 뉴스레터
- **카드뉴스 · SNS** → 인스타 비즈니스 계정 (조회·저장 많은 것)
- **유튜브** → 조회수 높은 영상의 썸네일·제목
- **PPT · 랜딩페이지** → SlideShare, 노션 공개 템플릿, 전환 잘 된 랜딩
- **보고서 · 기획안** → 상사에게 "통과된" 것
- **CS 응대** → 브랜드의 좋은 응대 메시지 캡처

### 얼마나 모을까
- **최소 3개, 이상적 5개**
- 너무 많으면 AI가 패턴을 흐리게 뽑음 (5개 전후가 가장 좋음)

### 주의사항
- "내가 좋다고 느낀 것"으로 모으세요
- 전문가들이 추천하는 거 말고, **본인 취향에 맞는 것**

---

## STEP 2 · 역설계 (AI에 패턴 뽑기 요청)

### 복붙 가능한 프롬프트

```
아래에 [결과물 종류]의 좋은 사례 [개수]개를 첨부합니다.
이 사례들의 공통 구조와 패턴을 뽑아주세요.

[사례 1]
(여기에 원문 붙여넣기)

[사례 2]
(여기에 원문 붙여넣기)

[사례 3]
(여기에 원문 붙여넣기)

---

분석 요청사항:
1. 오프닝(첫 문장 / 첫 화면) 방식
2. 전개 순서와 흐름
3. 자주 쓰이는 훅 · 키워드 · 표현
4. 문단 길이 · 호흡
5. 클로징(마지막) 방식
6. 이들이 공통적으로 쓰는 특이한 장치 · 트릭

각 항목별로 사례 인용과 함께 정리해주세요.
```

### 더 강하게 뽑고 싶을 때 (옵션)

AI가 너무 뻔한 답을 주면 이 한 줄 추가:

```
"뻔한 분석 말고, 제가 직접 분석했다면 놓쳤을 패턴을 뽑아주세요."
```

---

## STEP 3 · 이식 (내 변수로 교체)

### 복붙 가능한 프롬프트

```
위에서 뽑은 패턴을 바탕으로, 아래 주제에 맞춰 [결과물 종류]를 작성해주세요.

- 주제: [내 주제]
- 독자 / 타겟: [누구를 위한 것인가]
- 톤 · 무드: [원하는 느낌 — 예: 친근한, 전문적인, 유머러스한]
- 분량: [예: 3문단 / 500자 이내 / 10줄]
- 꼭 포함할 것: [핵심 메시지 · 키워드 · 사례]
- 피해야 할 것: [클리셰 · 특정 표현 · 형식]

위 조건에 따라, 앞서 뽑은 패턴을 충실히 적용해서 작성해주세요.
```

### 결과물이 아쉬울 때 (반복 개선)

```
"방금 작성한 것에서 [STEP 2에서 뽑은 패턴 중 특정 항목]이 약합니다.
그 부분을 더 살려서 다시 써주세요."
```

## 활용 예시 6가지

### 1. 블로그 · 뉴스레터 (문체 · 전개)
- 진단: 좋아하는 뉴스레터 최근 글 5편
- 역설계: 오프닝 · 문단 호흡 · 구어체 패턴
- 이식: 이번 주 내 주제로 패턴 적용

### 2. 카드뉴스 · SNS (훅 · 구성)
- 진단: 저장 많은 카드뉴스 5개
- 역설계: 1장 훅 · 슬라이드 순서 · 마지막 CTA
- 이식: 내 브랜드 콘텐츠 기획

### 3. 유튜브 (썸네일 · 제목)
- 진단: 내 채널 / 동종 채널 조회수 높은 영상 10개
- 역설계: 제목 구조 · 썸네일 텍스트 배치 · 감정 단어
- 이식: 다음 영상 제목 · 썸네일

### 4. PPT · 랜딩페이지 (레이아웃 · CTA)
- 진단: 전환 잘 된 랜딩 3개 + SaaS 대시보드 5개
- 역설계: 섹션 순서 · CTA 배치 · 헤드라인 포맷
- 이식: 내 서비스 랜딩 구조

### 5. 보고서 · 기획안 (구조 · 근거 배치)
- 진단: 내 팀에서 "통과된" 보고서 3~5개
- 역설계: 결론 위치 · 근거 순서 · 데이터 제시 방식
- 이식: 이번 주 보고서 작성

### 6. CS · 고객 응대 (말투 · 해결 플로우)
- 진단: 잘한 CS 응대 캡처 5개 + 못한 것 2개 (반면교사)
- 역설계: 공감 → 원인 → 해결 → 다음 액션 흐름, 호칭, 마무리 문장
- 이식: 이번 주 CS 이슈 응대

## 자주 하는 실수 · 피하세요

| 실수 | 이유 | 대안 |
|---|---|---|
| 사례 1~2개만 모으기 | AI가 너무 편향된 패턴 뽑음 | 최소 3개, 5개 이상이면 이상적 |
| "잘 써줘" 같은 모호한 이식 요청 | AI가 일반적인 답만 냄 | 주제·독자·톤·분량 구체적으로 지정 |
| 한 번에 완벽한 결과 기대 | 첫 결과는 보통 70점 | 반복 개선 질문으로 90점까지 올리기 |
| 패턴만 보고 내 맥락 무시 | 복붙 결과물이 됨 | 내 변수(상황·독자·브랜드)로 항상 교체 |

## 봇으로 만들기 (반복 쓸 때)

이 3단계 프롬프트가 자주 쓰인다면 봇으로 저장하세요.

1. AI랑 대화로 만족스러운 역설계 1회 완성
2. AI에게: "이 과정을 봇으로 만들게. 지시문 써줘"
3. AI가 지시문 뽑아줌
4. GPTs / Projects / Gems에 붙여넣기
5. 다음부터 클릭 한 번으로 실행
